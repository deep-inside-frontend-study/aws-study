## 0. AWS 실습 환경 구축하기

### AWS
- 아마존이 제공하는 클라우드 컴퓨팅 플랫폼
- 인터넷을 통해 서버, 저장소, 디비, 네트워크 등 IT 인프라를 필요한 만큼 빌려쓸 수 있는 서비스

#### 클라우드 컴퓨팅
- 직접 서버를 사서 운영하는 것이 X라, 아마존의 데이터센터에 있는 컴퓨터 자원을 인터넷으로 빌려 쓰는 것
- 주요 서비스
  - EC2 : 가상 서버
  - S3 : 파일 저장소
  - RDS : 디비
  - Lambda : 서버를 항상 켜두지 않고, 특정 이벤트가 발생할 때만 코드를 실행하는 서비스
  - CloudFront : 전 세계 곳곳에 캐시 서버를 두고, 사용자와 가장 가까운 서버에서 콘텐츠를 전달하는 서비스 (CDN)

### AWS 사용법
#### 레벨별 AWS 사용법
- 레벨 1
  - 사용자 인터페이스 사용이 가능한 AWS 관리 콘솔
  - UI를 통한 직관적인 AWS 사용이 가능 BUT 생성한 AWS 리소스가 늘어날수록 관리하기 힘들어진다는 단점
- 레벨 2
  - AWS SDK와 AWS CLI로 스크립트와 터미널을 이용하여 AWS 사용
  - AWS SDK
    - 자바, js, 파이썬 등 자주 사용되는 언어를 바탕으로 AWS를 호출하여 웹, 앱 애플리케이션 구축 도움
- 레벨 3
  - IaC (Infrastructure as Code) 툴인 클라우드포메이션을 이용하여 JSON, YAML 형식으로 클라우드 인프라 정의
  - 다수의 AWS 리소스를 생성/관리 가능
- 레벨 4
  - AWS CDK (Cloud Development Kit) 
  - 프로그래밍 언어를 통해 클라우드 인프라를 정의하는 고급 도구 (여러 기능적 요구사항 필요)

### VSCode로 클라우드포메이션 템플릿 관리하기
우선 레벨 3의 클라우드 포메이션 사용 환경을 VSCode(AWS 클라우드포메이션 탬플릿 관리 도구)로 구축하겠다.

- VSCode를 사용하면 **클라우드포메이션 린터** (클라우드포메이션 확장 프로그램) 사용 가능
  - JSON, YAML 문법을 검사하여 클라우드포메이션 템플릿 형식을 유지하도록 도와줌

#### 아마존 VPC 생성하기
- **아마존 VPC** (Amazon Virtual Private Cloud)
  - 클라우드의 네트워크 환경을 구축하고 관리할 수 있는 서비스
- **스택** : 클라우드포메이션 템플릿은 AWS에서 스택이라는 이름으로 생성되어 관리됨
