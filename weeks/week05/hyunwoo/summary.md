### 13장

- ECS는 도커 컨테이너를 띄우고 관리할 수 있게 해주는 서비스
- 도커는 아실 것 같아서 설명은 생략하겠습니다.
- 작업 정의 / 작업 / 서비스 / 클러스터로 이루어져 있다.
- 컨테이너 1개로 무중단 배포 가능한가?

```
minimumHealthyPercent = 100
maximumPercent = 200
desiredCount = 1
```

이렇게 두면 배포시에 일시적으로 컨테이너 2개 뜨지만, 결과적으로 실행중인 컨테이너 1개로도 무중단 배포가 가능하게 할 수 있다.

```
1. 평소
old task 1개

2. 배포 시작
old task 1개 + new task 1개

3. new task health check 통과
old task 1개 + new task 1개

4. old task 제거
new task 1개

5. 배포 완료
총 1개
```

### 14장

- 람다는 넓은 의미에서 서버리스, 좁은 의미에서 FasS이다.
- 코드 실행 시간과 요청 수에 따라 비용이 청구된다.
- 가끔 호출된다면 콜드 스타트가 발생하기 때문에 가벼운 요청, 웹훅 등에 적합하고 실시간성이 중요한 작업에는 단독으로 어울리지 않는다.
- 스케줄러 등에 활용해보면 좋겠다는 생각을 하지만 사실 EventBridge 등과 같은 더 특화된 제품들이 있어서 정확히 어디에 쓰면 좋을까? 라는 생각을 한다.
