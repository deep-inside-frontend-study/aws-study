## 6장 S3

- 객체 스토리지 -> 비정형 형식의 데이터를 저장하기 때문에 아무거나 다 가능
- 버킷 용량은 무제한, 객체는 0바이트에서 ~ 5TB까지

### 스토리지 클래스

- standard
  - 자주 접근하는 데이터
  - 여러 AZ에 저장 → 매우 높은 내구성
  - 빠른 접근 속도
- Infrequent Access(standard - IA)
  - 가끔 쓰는 데이터
- One Zone I-A

  - 더 싸지만 위험 있음
  - 한 AZ에만 저장됨
  - 장애 나면 데이터 날아갈 수도 있음

- Glacier (Instant / Flexible / Deep Archive)

  - Glacier Instant → 바로 접근 가능 (하지만 저렴)
  - Glacier Flexible → 몇 분~몇 시간 걸림
  - Glacier Deep Archive → 몇 시간~하루 걸림 (가장 쌈)

  - 저장 비용 매우 낮음
  - 대신 꺼낼 때 느림 + 비용 있음

- Intelligent-Tiering
  - 자동으로 클래스 바꿔주는 옵션 (요금최적화?)
  - 예측 불가능한 데이터

### 버저닝

- 버저닝 설정하면 단일 객체의 여러버전 사용 유지 가능
- 설정 안하면 단일 객체, 삭제, 덮어쓰기하면 날라감
- 버저닝하면 그만큼 사용료도 늘어남

### 생명주기

- 생명주기 설정으로 자동 삭제 가능
- 최대 int최대값까지가능 (왜 sint최대값으로?)

## 7장 프론트엔드를 위한 서비스

### route53

- 도메인 관리를 위한 서비스
- 도메인 구매, 트래픽 전달

#### FQDN (fully qualified domain name)

- 문자열로 ip를 대체
- 도메인과 호스트를 포함
- 호스트는 생략가능

- DNS, DNS서버

  - dns -> ip <-> 도메인
  - OS 캐시 확인
  - DNS Resolver (재귀 DNS 서버) 보통 ISP나 회사 DNS (8.8.8.8)
  - Root DNS 서버
    - .com 어디서 관리하는지 알려줌 → .com은 저쪽 서버(TLD)에 있는거 알려줌
  - TLD DNS 서버 (.com)
  - Authoritative DNS 서버
    - 진짜 IP 알고 있는 서버
  - 없으면 404

- 프라이빗 도메인도있음 (내부 vpc접근)
