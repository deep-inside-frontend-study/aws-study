## emplify

- 간편하게 배포하는 용도
- Next.js SSR 앱 배포도 지원
- db는 다이나모 db랑 연동됨
  - ```
    import { type ClientSchema, a, defineData } from '@aws-amplify/backend';
    ```
  - 이런식으로 사용한다고함
  - 로직은 lambda이용
- 인증은 Cognito, 파일은 S3
- redis류의 캐싱은 Amazon ElastiCache

- 가격 차이

## cognito

- 사용자관리와 인증 기능을 제공
- 사용자 풀(user pool)

- 자격증명 풀 (identify pool)
  - aws 리소스에 대한 접근 권한 부여

## 백엔드 서비스 이해하기

- 탄력적 로드 밸런서 (ELB)
  - 퍼블릭 서브넷에 배치
  - 인스턴스에 domain을 할당하는게 아니라 로드밸런서에 할당
    - 이러면 route53은 elb를 가리키는게 될듯?
  - ALB (application)
    - 응용계층을 사용해 부하분산
    - http, https 트래픽 라우팅
  - NLB (network)
    - tcp, udp 트래픽 라우팅
  - GWLB (gateway)
    - 타사 제품을 aws로 마이그레이션하는데사용 ?
    - 방화벽이나 보안 장비를 중간에 끼워 넣기 위한 로드밸런서
    - 보안 장비/네트워크 검사 구조를 AWS 방식으로 마이그레이션할 때 쓰는 서비스

### 로드 밸런서 동작 방식

- 퍼블릭 / 프라이빗에 생성할지 선택
- 여러 가용 영역에 걸쳐 부하 분산
  - 각 가용영역에 부하 분산을 도와주는 "로드 밸런서 노드"생성
- 보안그룹으로 사용자 가려받기 가능
- 그림 예시로, 로드밸런서 하나로 프라이빗, 퍼블릭 둘다 커버하는건 불가능함

### 대상 그룹

- 가용 영역 안에 있는 ec2 인스턴스는 어떻게 구분?
  - 대상그룹으로 지정
  - health 체크, 80, 443 포트 둘다 가능

### 리스너

- 프로토콜과 포트를 지정하고, 대상 그룹을 연결
- ex: 443 -> A그룹, 444 -> B그룹
- 리버스프록시 같은 느낌인듯?

### 고정 응답 반환 규칙

- 특정 경로에 접근했을 때 사용자에게 응답코드, 텍스트 반환 가능
- html, javascript, json 선택해서 반환 가능

### 로드 밸런서 알고리즘

- round robin method
  - 순서대로 할당하는 방식
  - 첫번째 요청은 첫번쨰 서버, 두번째는 두번째 ...
- weight random method
  - 대상 서버의 장애, 성능 저하 감시
- least outstanding method (최소 미해결 요청)
  - 연결수가 가장 적은 서버로 라우팅
